Detailed Description

Piccolo OS overrides for Pi Pico SDK lock_core defaults.

Author
Keith Standiford
Version
1.0
Date
2022-08-29
Copyright
Copyright (c) 2022 Keith Standiford

This file contains specific definitions and parameters to override default values in the Pi Pico SDK in order to provide compatibility with the Piccolo_Os cooperative multi-tasking operating system originally written by Gary Sims. It should be included in CMakelists.txt files by adding the following line before the pico_sdk_init() but after the project(...) line:

list(APPEND PICO_CONFIG_HEADER_FILES ${PROJECT_SOURCE_DIR}/piccolo_os_lock_core.h)

◆ lock_owner_id_t

#define lock_owner_id_t   int32_t

type to use to store the 'owner' of a lock.By default this is int8_t as it only needs to store the core number or -1, however it may be overridden if a larger type is required (e.g. for an RTOS task id)

For Piccolo OS, it will hold a task identifier (core number or task structure address)
